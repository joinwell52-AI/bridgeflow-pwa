# TASK-20260511-001：BUG-SDK-007 hotfix — `Agent.create()` 不传 model（v0.2.0-beta.3）

## 一、TL;DR

QA-014 smoke 揭示：**ADMIN key + `Agent.create({ model: { id: "X" } })` 100% 失败**（X = `default` / `claude-sonnet-4` / `claude-sonnet-4-5` 均 fail），但 ADMIN key + `Agent.create()` 不传 model 100% 成功（QA-011 + QA-014 no-model 路径双重确认）。MT-1 引入的「`Agent.create` 路径传 model」在 ADMIN key 下被 Cursor 后端按 key 订阅级别拒绝。

**根因（推断）**：Cursor 后端按 API key 订阅级别对「allow programmatic model specification on Agent.create」字段有 ACL 白名单；DEV-13 自测用 DEV key 成功 = key 差异；ADMIN key 不在白名单内。

**修复目标**：解除 `Agent.create()` 阶段的 model 参数 → 仅在 `agent.send()` 阶段传 model，让 ADMIN key 路径解锁，同时不 break TS-MODEL-1~5 / TS-RUN-1~2 现有契约。

**出厂版本**：`v0.2.0-beta.3`（MT-5 single hotfix）。

## 四、修复方向（DEV 自选 + 推荐方向 A）

### 方向 A（QA 推荐 + PM 推荐 + 默认）— `Agent.create()` 不传 model

- `Agent.create()` 永远**不传** `model` 参数（即使 `modelId` 存在）
- `agent.send()` 保留 `model` 参数传递（不变）
- 文档（JSDoc + .env.example）更新：`CURSOR_DEFAULT_MODEL` **不再**影响 `Agent.create()`，仅在 `send()` 时生效
- 若 SDK 在 `Agent.create()` 缺 model 时报「Local SDK agents require an explicit model」（BUG-SDK-001 原始错误）→ **回到 BUG-SDK-001 状态**（不应该，因为 QA-011/014 no-model 路径已证明 Agent.create no-model 在 ADMIN key 下成功；DEV 需在 smoke 中实证「Agent.create no-model 后 send with model 的完整链路」是否端到端通）

**风险点**：若 `agent.send()` 也被 ADMIN key 拒绝 model 参数 → 落入方向 B。

### 方向 B（保留 fallback）— 完全不传 model

- `Agent.create()` 和 `agent.send()` 都不传 model
- 让 SDK 内部默认行为接管
- 退回到 v0.2.0-beta（pre-MT-1）的 model 路径，但保留 MT-2/3/4 的其它修复
- `CURSOR_DEFAULT_MODEL` 仅作为 `.env.example` 文档项，**运行时不读**（或读但仅用于 banner 显示）
- DEV 自由调整 TS-MODEL-1~5 测试期望（部分测试可能改为 `model: undefined` 断言）

**风险点**：BUG-SDK-001 修复目标（per-task model override）需要重新规划。

### 方向 C（环境开关）— 按 key 类型分流

- 加 `CURSOR_SDK_MODEL_ON_CREATE=true|false`（默认 `false`）
- 默认行为 = 方向 A
- DEV 持有 enterprise key 时手动开 `true` 走 with-model 路径
- 复杂度高，但兼顾两类 key

**PM 倾向**：先方向 A，若方向 A 在 smoke 中验证发现 `send()` 也拒绝 model → 自动滑到方向 B；不上方向 C（过度工程）。
