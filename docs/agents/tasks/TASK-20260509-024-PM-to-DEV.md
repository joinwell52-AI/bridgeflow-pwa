## 主交付（5 件）

### 主交付 1：`SkillRegistry`（`packages/codeflow-runtime/src/skill/SkillRegistry.ts`，新文件）

加载 + 索引 skill 注册表。每个 skill 落 `<skillsDir>/<skill_id>.json`，front-matter 必须严格符合 [`packages/codeflow-protocol/schemas/skill.schema.json`](../../packages/codeflow-protocol/schemas/skill.schema.json)。

```ts
export interface SkillRecord {
  skill_id: string;
  version: string;
  provided_by: { type: "mcp_server"; transport: "stdio" | "http" | "sse"; command?: string; url?: string };
  tools: ToolSpec[];
  available_to_roles: string[];
  required_kernel: string[];                 // ["fcop@>=1.0"] 强约束（schema "contains" 已校验）
  compatible_runtimes?: ("local" | "cloud")[];
  // ... displayName / homepage / license
}

export class SkillRegistry {
  constructor(opts: { skillsDir: string; logger?: Logger });

  /** 启动时扫描并加载所有 skill。校验失败的文件 logger.warn 跳过（不阻塞其他 skill）。*/
  async load(): Promise<{ loaded: SkillRecord[]; skipped: { file: string; reason: string }[] }>;

  /** O(1) 查找 */
  getById(skill_id: string): SkillRecord | null;

  /** 按 role 反查可用 skills */
  listForRole(role: string): SkillRecord[];

  /** 列举所有 */
  list(): SkillRecord[];
}
```

**实现要点**：
- 用 `yaml@^2`（Phase C 已加依赖）+ `JSON.parse`（skill 文件用 .json 后缀，按 schema $id 命名）
- 用 atomic-read（`fs.promises.readFile` + retry on EBUSY）— 复用 Phase A `_internal/atomic-write` helper 的 retry pattern
- schema 校验 = 用 `@codeflow/protocol` 包导出的 `validate("skill", obj)` API（如不存在则 throw）
- `getById` / `listForRole` 用 in-memory Map + Set 索引

### 主交付 2：`KernelDependencyValidator`（`packages/codeflow-runtime/src/skill/KernelDependencyValidator.ts`，新文件）

启动时校验：每个 agent 的 skills 集合必须包含至少一个 `required_kernel: ["fcop@..."]` 的 skill。**缺 fcop 拒绝加载该 agent**。

```ts
export interface ValidationFailure {
  agent_id: string;
  reason: "no_fcop_skill" | "skill_not_found" | "no_compatible_runtime";
  detail: string;
}

export class KernelDependencyValidator {
  constructor(opts: { skillRegistry: SkillRegistry; logger?: Logger });

  /** 对单个 agent 的 skills 列表做校验 */
  validateAgent(agent: AgentRecord): ValidationFailure | null;

  /** 对全部 agents 做校验，返回所有失败 */
  validateAll(agents: AgentRecord[]): ValidationFailure[];
}
```

**校验规则**（按 §0.5 + §0.7.5 + skill.schema.json `required_kernel.contains: "^fcop@.+"`）：

1. **必须包含 fcop**：`agent.skills` 中至少一个 skill 的 `required_kernel` 含 `^fcop@.+`（pattern 跟 schema `contains` 一致）
2. **skill 存在**：`agent.skills` 列表里所有 skill_id 必须能在 SkillRegistry 找到
3. **runtime 兼容**：v0.1 强制 `compatible_runtimes` 包含 `"local"`（v0.1 不做 cloud agent，按 §0.8.2 line 2342-2347 显式不做清单）

校验失败 = 写入 `RuntimeBootstrap.report.failed`，**不**抛异常（让其他 agent 继续启动）。在 `report` 里清晰标记哪些 agent 因为 kernel 校验失败被跳过。

### 主交付 3：`MCPInjector` stub（`packages/codeflow-runtime/src/skill/MCPInjector.ts`，新文件）

按 agent.role 把对应 skills 的 MCP 配置注入到 SDK 实例。**v0.1 = stub 接口**，等 v0.2 接真实 `@cursor/sdk` MCP runtime 时实化。

```ts
export interface MCPMount {
  agent_id: string;
  skill_id: string;
  transport: "stdio" | "http" | "sse";
  command?: string;
  url?: string;
}

export class MCPInjector {
  constructor(opts: {
    skillRegistry: SkillRegistry;
    sdkAdapter: AgentSdkAdapter;
    logger?: Logger;
    /** v0.1 默认 "stub" — 不实际 spawn MCP server，只做日志记录；v0.2 改 "live" 接真实 SDK */
    mode?: "stub" | "live";
  });

  /** 给 1 个 agent 装上它能用的全部 skills 的 MCP servers */
  async mount(agent: AgentRecord): Promise<MCPMount[]>;

  /** 卸载（agent dispose 时） */
  async unmount(agent_id: string): Promise<void>;
}
```

**v0.1 stub 行为**：
- `mount` 只 logger.info(`[MCPInjector stub] mounting ${count} skills for agent ${agent_id}: ${skill_ids.join(", ")}`)，**不 spawn 子进程**
- 返回 `MCPMount[]` 数组让 caller 知道哪些 skill 应当被装上
- `mode: "live"` v0.1 直接 throw `MCPInjectorLiveModeNotImplementedError`（同 NeedsHumanGate 决策 O 的 eager-throw 模式）

### 主交付 4：Runtime 集成 + AgentRegistry.register hook

#### 4.a `RuntimeBootstrap` 加 kernel 校验阶段

在 `Bootstrap.run()` 流程中插入：

```
1. PersistentStore.loadAll() → agents
2. SDK.list() → sdkAgents
3. reconcile(agents, sdkAgents)              ← 既有 Phase A 逻辑
4. **新增**：kernelValidator.validateAll(reconciled.success)
   - 失败的 agent 从 success → failed，附 reason="no_fcop_skill" / "skill_not_found"
5. **新增**：mcpInjector.mount(agent) for each surviving success agent
6. return ReconciliationReport（含新字段 kernel_failures: ValidationFailure[]）
```

`ReconciliationReport` 接口加 `kernel_failures: ValidationFailure[]`（在 `src/types/state.ts` 或 `src/registry/types.ts`）。

#### 4.b `AgentRegistry.register` 加前置 hook（不破坏接口）

通过依赖注入（在 Runtime composition root）让 register 在 store.upsert 之前调 KernelDependencyValidator。失败 → throw 新错误类 `KernelDependencyError`（co-located 在 errors.ts，沿决策 J）。

#### 5.b Runtime composition root

```ts
const skillsDir = join(persistDir, "skills");          // 默认 .codeflow/state/skills/
```
