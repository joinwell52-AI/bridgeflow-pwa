### 主交付 1：`ReviewWriter`（`packages/codeflow-runtime/src/review/ReviewWriter.ts`，新文件）

把 review verdict 落成 `REVIEW-{date}-{seq}-{REVIEWER}-on-TASK-{date}-{seq}.md` 文件，front-matter 严格符合 [`packages/codeflow-protocol/schemas/review.schema.json`](../../packages/codeflow-protocol/schemas/review.schema.json)。

```ts
export interface ReviewVerdict {
  review_id: string;                   // REVIEW-YYYYMMDD-NNN-XXX-on-TASK-...
  subject_type: "task" | "code_change" | "report" | "role_switch";
  subject_ref: string;                 // 如 task_id
  reviewer_role: string | null;
  reviewer_agent: string | null;
  decision: "approved" | "rejected" | "needs_changes" | "abstained" | "needs_human";
  rationale?: string;
  required_changes?: string | string[]; // decision=needs_changes 必填
  human_approval?: HumanApproval;       // decision=needs_human 必填
  decided_at: string;                   // ISO-8601
  decision_duration_ms?: number;
}

export interface HumanApproval {
  pushed_to: "mobile" | "cli";          // v0.1 始终 "cli"
  pushed_at: string;
  approved_by?: string | null;
  approved_at?: string | null;
  trigger_reason: string;
}

export class ReviewWriter {
  constructor(opts: { reviewsDir: string });
  async write(verdict: ReviewVerdict, body: string): Promise<string>;  // 返回 filepath
}
```

**实现要点**：
- 文件名格式严格匹配 `^REVIEW-\d{8}-\d{3}-[A-Z]+-on-TASK-\d{8}-\d{3}.*\.md$`（review.schema.json `review_id` pattern）
- front-matter YAML 用 `yaml` 包（Phase C 已加依赖）
- body markdown 部分包含 rationale + reviewer notes
- `additionalProperties: true` schema 允许扩展，但 v0.1 只写 schema 定义字段
- 用 atomic-write helper（Phase A 已有 `_internal/atomic-write.ts`）

`decision=needs_human` 时触发的网关。v0.1 = stdout 输出，v0.2+ = Mobile push。

```ts
export interface HumanPushRequest {
```

### 主交付 3：`ReviewEngine`（`packages/codeflow-runtime/src/review/ReviewEngine.ts`，新文件）

v0.1 review 主循环：

```ts
export interface ReviewPolicy {
  /** 决定 task 完成时是否触发 review。默认全部触发。*/
  shouldReview(taskRef: TaskReference): boolean;
  /** 决定 reviewer role。默认按 task.recipient 选对应 reviewer 角色 */
  pickReviewer(taskRef: TaskReference): string;
}

export class ReviewEngine {
  constructor(opts: {
    sessionManager: SessionManager;
    registry: AgentRegistry;
    historyWriter: StateHistoryWriter;
    reviewWriter: ReviewWriter;
    needsHumanGate: NeedsHumanGate;
    policy?: ReviewPolicy;                 // 默认 DefaultReviewPolicy
    reviewsDir: string;                    // .codeflow/reviews/
    logger?: { info(...); warn(...); error(...) };
  });

  start(): void;       // 订阅 SessionManager.onEvent('runtime.session_ended')
  stop(): void;        // unsubscribe
}
```

**Review 流程**（订阅 `runtime.session_ended`）：

```
1. session ended → 找 task_ref（payload.task_id）
2. policy.shouldReview(taskRef)
   - false → state_history append "from: ended / to: review_skipped" → 退出
   - true → step 3
3. policy.pickReviewer(taskRef) → 找 reviewer agent in registry
   - 找不到 → state_history "to: reviewer_not_found" → 触发 NeedsHumanGate.push（兜底）
   - 找到 → step 4
4. 派 reviewer 跑 review subtask
   - **v0.1 简化**：不再 chokidar 派 TASK-...-PM-to-REVIEWER.md（避免环路）
   - 而是直接 SessionManager.startSession(reviewer_agent_id, review_subtask_id, payload)
   - payload 含被 review 的 task_id + body 摘要
5. 收 reviewer session 终结 → 解析 reviewer agent stdout 取 verdict（v0.1 用约定 stdout 格式 "VERDICT: <decision>; RATIONALE: <text>"；v0.2 改 reviewer 写 REVIEW-*.md 自带）
   - 解析失败 → fallback decision="needs_human" + trigger_reason="verdict_parse_failed"
6. ReviewWriter.write(verdict, body) → 落 REVIEW-*.md
7. 如 decision="needs_human" → NeedsHumanGate.push() → 把 HumanApproval stub 写回 verdict.human_approval
8. state_history append "from: review_pending / to: review_<decision>" 到原 task.md
```

**注意 — 决策 B' 兜底（顺手做）**：

按 [`REPORT-20260509-018-DEV-to-PM.md`](./REPORT-20260509-018-DEV-to-PM.md) §五决策 B'，`SessionManager.startSession` 成功**不**主动改 agent.status="running"。S4 在 ReviewEngine 实施时**顺手补这个钩子**，让 doorbell 路径的 reject_busy 真正能触发：

- 监听 `runtime.session_started` → 调 `registry.upsert(agent, { status: "running" })`
- 监听 `runtime.session_ended` / `_cancelled` / `_failed` → 调 `registry.upsert(agent, { status: "idle" })`
- 这是 v0.1 集成层钩子，**不**改 SessionManager 接口（保持 Phase B 决策稳定）
- 命名：`AgentStatusReconciler`（新文件 `packages/codeflow-runtime/src/registry/AgentStatusReconciler.ts`）
- 在 Runtime composition root 装配（同 ReviewEngine 装配位置）

如 QA-021 工作 3 推荐**不**在 S4 做（推荐留 v0.2），DEV 在动手前**先停手等 PM**——这是 PM 与 QA 的协作依赖，不是 DEV 自决范围。
