### 必交付 3：补全 §3.4 SessionManager / SessionStore / TranscriptWriter 5 个测试场景的"通过标准"细节

[`REPORT-010 §3.4`](./REPORT-20260509-010-QA-to-PM.md) 已经设计了 TS-4.1~TS-4.5 的标题，但**通过标准是占位**（"Phase B 待补全"）。本任务期间——**与 DEV Phase B 实施并行**——QA 把它们填实：

#### TS-4.1 SessionStore 单 record per file 落盘

输入：`SessionStore.save(sessionRecord)` with `session_id = "sess-xxx"`

操作步骤：
1. 构造 fixture sessionRecord
2. 调 `save(sessionRecord)`
3. 检查文件系统

期望：
- `<dir>/sess-xxx.json` 文件存在
- 文件内容 = `JSON.stringify(sessionRecord, null, 2)` 等价
- `<dir>/sess-xxx.json.tmp` 不可见（atomic rename 已完成）

通过标准：
- `await fs.access(path)` 不抛
- `JSON.parse(await fs.readFile(path, "utf-8"))` deepEqual sessionRecord
- 目录里没有 .tmp 文件

#### TS-4.2 TranscriptWriter append-only 单事件追加

输入：`writer.attach(runId, mockHandle)` + 触发一个 `RuntimeEvent`

操作步骤：
1. 构造 `MockRunHandle`（spike 模式或 EventEmitter 模拟）
2. attach
3. 触发 1 个 `message_delta` 事件
4. 读 transcript 文件

期望：
- `<dir>/<run_id>.md` 1 行新增
- 行格式：`[ISO timestamp] [message_delta] payload_summary`

通过标准：
- 文件存在且行数 = 1
- 行匹配正则 `/^\[\d{4}-\d{2}-\d{2}T.*Z\] \[message_delta\] /`

#### TS-4.3 TranscriptWriter 高频 1000 事件流不丢事件

输入：connect + 触发 1000 次 `message_delta`，间隔 0ms

期望：
- 文件行数 = 1000（**不少**——丢事件 = fail）
- 每行格式正确

通过标准：
- `(await fs.readFile(path, "utf-8")).split("\n").filter(Boolean).length === 1000`
- 整个测试 < 5 秒（不能过慢）

**注意**：DEV 实现可能用 `createWriteStream` + buffer，需要在测试 close 前 `await writer.close(runId)` 才能保证 flush；本场景 assertion 必须在 close 之后。

#### TS-4.4 SessionStore 元数据更新 → atomic-write

同 PersistentStore TS-1.1 / TS-1.3 的逻辑，但 per-file（不是 saveAll）。

输入：先 `save(record)`，然后修改 `record.status = "completed"` 后再 `save(updated)`，期间 mock `fs.rename` throw 一次

期望：第一次 save 后文件存在；第二次 save 失败时**原文件保持第一次内容**（atomic rename 的 durability）

通过标准：
- 第二次 save 抛错
- 之后 `await load(session_id)` 返回**第一次 save 的版本**

#### TS-4.5 跨 run 累计 cost 字段正确累加

输入：构造 SessionRecord 含 `runs: [{ tokens_used: ..., cost: 0.05 }, { tokens_used: ..., cost: 0.07 }]`，初始 `total_cost_usd = 0`

期望：调 SessionManager 内的某个 cost 累加 helper（如果存在；如果 DEV 未实现 helper，则在 SessionRecord 写入时显式累加）

通过标准：
- 累加后 `record.total_cost_usd === 0.12`
- 浮点容差 `Math.abs(record.total_cost_usd - 0.12) < 1e-9`
