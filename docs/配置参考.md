# BridgeFlow 配置参考

## 概述

BridgeFlow 由两部分配置驱动：

| 配置文件 | 位置 | 作用 |
|----------|------|------|
| `bridgeflow_config.json` | PC 端工作目录 | PC 执行机全部运行参数 |
| `web/pwa/config.js` | PWA 前端 | 手机端中继地址与版本 |

---

## 一、PC 端配置文件 `bridgeflow_config.json`

### 生成方式

```powershell
bridgeflow init                                  # 生成默认配置
bridgeflow init --relay-url wss://... --room-key xxx   # 指定中继/房间
bridgeflow init --force                          # 强制覆盖已有配置
```

配置文件生成在**当前工作目录**，也可以通过 `--project-root` 指定目录：

```powershell
bridgeflow init --project-root D:\my-project
```

---

### 完整字段说明

#### `relay` — 中继连接

```json
"relay": {
  "url": "wss://ai.chedian.cc/bridgeflow/ws/",
  "room_key": "bridgeflow-default",
  "shared_secret": ""
}
```

| 字段 | 说明 | 是否必改 |
|------|------|----------|
| `url` | 中继 WebSocket 地址，默认使用公共中继 | 自部署时改 |
| `room_key` | 房间隔离 key，同一房间的 PC 和手机才能通信 | **建议改成随机字符串** |
| `shared_secret` | 可选的共享密钥（预留，当前未强制） | 选填 |

> **room_key 安全建议**：默认值 `bridgeflow-default` 是公开房间，任何人都可以连入。团队使用时请改为随机字符串，例如 `team-xyz-2026`。

---

#### `device` — 设备标识

```json
"device": {
  "device_id": "my-pc",
  "device_name": "我的 AI 执行机",
  "owner_role": "PM01",
  "machine_code": "BF-XXXX"
}
```

| 字段 | 说明 | 填写方式 |
|------|------|----------|
| `device_id` | PC 的唯一标识，用于中继路由 | `bridgeflow init` 自动生成（主机名+slug） |
| `device_name` | 显示名称，显示在仪表盘 | 自动生成，可手动改 |
| `owner_role` | 本机主要负责的 AI 角色 | `PM01` / `DEV01` / `OPS01` / `QA01` |
| `machine_code` | 绑定用机器码，格式 `BF-XXXXXXXX` | `bridgeflow init` 自动生成（不要手动改） |

---

#### `bind` — 设备绑定状态

```json
"bind": {
  "status": "unbound",
  "bound_mobile_device_id": "",
  "bound_mobile_device_name": "",
  "bound_at": "",
  "bind_code_ttl_seconds": 600
}
```

| 字段 | 说明 |
|------|------|
| `status` | `unbound` / `pending` / `bound`，由 CLI 命令自动维护 |
| `bind_code_ttl_seconds` | 绑定码有效时长（秒），默认 600 秒 |
| 其余字段 | 绑定后自动填入，不需要手动填 |

---

#### `project` — 项目目录

```json
"project": {
  "root_dir": ".",
  "tasks_dir": "docs/agents/tasks",
  "reports_dir": "docs/agents/reports",
  "issues_dir": "docs/agents/issues"
}
```

| 字段 | 说明 |
|------|------|
| `root_dir` | 项目根目录（`bridgeflow init` 自动设为工作目录的绝对路径） |
| `tasks_dir` | 任务文件写入目录（相对 `root_dir`） |
| `reports_dir` | 回执文件写入目录 |
| `issues_dir` | 问题记录目录 |

---

#### `runtime` — 运行态目录

```json
"runtime": {
  "runtime_dir": ".bridgeflow/runtime",
  "idle_seconds": 180,
  "stale_task_seconds": 900
}
```

| 字段 | 说明 |
|------|------|
| `runtime_dir` | 运行时状态文件目录（已 gitignore） |
| `idle_seconds` | 角色超过多少秒无动作判定为"空闲"，默认 180s |
| `stale_task_seconds` | 任务超过多少秒无回复判定为"过期"，默认 900s |

---

#### `admin` — 真人管理员

```json
"admin": {
  "sender": "ADMIN01",
  "target": "PM01",
  "default_priority": "P1"
}
```

| 字段 | 说明 |
|------|------|
| `sender` | 真人角色 ID（固定 `ADMIN01`） |
| `target` | 默认接收任务的 AI 角色（通常是 `PM01`） |
| `default_priority` | 默认任务优先级：`P0`（紧急）/ `P1`（高）/ `P2`（中）/ `P3`（低） |

---

#### `ai_team` — 团队角色配置

```json
"ai_team": {
  "fixed_roles": ["ADMIN01", "PM01", "DEV01", "QA01"],
  "cursor_only": true
}
```

| 字段 | 说明 |
|------|------|
| `fixed_roles` | 团队参与角色列表 |
| `cursor_only` | 是否只在 Cursor 中运行（当前固定 `true`） |

---

### 运行时更新配置

配置已存在时，`bridgeflow init` 只更新指定字段，不会覆盖其他内容：

```powershell
# 只换中继地址
bridgeflow init --relay-url wss://new-relay/bridgeflow/ws/

# 只换房间
bridgeflow init --room-key my-new-room

# 同时换
bridgeflow init --relay-url wss://new-relay/ws/ --room-key my-room
```

或者直接编辑 `bridgeflow_config.json` 中的 `relay` 块，修改后重启 `bridgeflow run` 生效。

## 二、PWA 配置文件 `web/pwa/config.js`

手机端 PWA 通过此文件读取中继连接参数。

```js
global.BRIDGEFLOW_CONFIG = {
  appName:      "BridgeFlow",
  appVersion:   "1.6.0",          // Service Worker 缓存版本，改动后必须递增
  relayUrl:     "wss://ai.chedian.cc/bridgeflow/ws/",
  relayLabel:   "公共中继",
  roomKey:      "bridgeflow-default",
  autoConnect:  true,             // 打开 PWA 是否自动连接中继
  defaultTarget: "PM"             // 任务列表默认显示的角色
};
```

| 字段 | 说明 | 注意 |
|------|------|------|
| `appVersion` | 每次修改 PWA 文件后必须递增 | 否则 Service Worker 不更新，手机看到旧版 |
| `relayUrl` | 中继地址，与 PC 端保持一致 | 必须同一 relay |
| `roomKey` | 房间 key，与 PC 端保持一致 | 必须同一 room_key |
| `autoConnect` | `true` 打开即连，`false` 手动点连接 | 建议 `true` |

> **PWA 配置修改后必须同步推送到 GitHub Pages**，手机才能获取新版。

## 三、`bridgeflow init` 参数速查

```
bridgeflow init [options]

选项：
  --project-root PATH    项目根目录，默认为当前目录
  --relay-url URL        中继地址，覆盖默认值
  --room-key KEY         房间 key，覆盖默认值
  --force                强制覆盖已有配置（⚠️ 会重置绑定状态）
```

---

## 四、`bridgeflow run` 参数速查

```
bridgeflow run [options]

选项：
  --config PATH    配置文件路径，默认为当前目录下的 bridgeflow_config.json
```

示例：

```powershell
# 使用当前目录的配置
bridgeflow run

# 指定配置文件（多项目时使用）
bridgeflow run --config D:\project-a\bridgeflow_config.json
```

## 五、典型场景配置

### 场景 1：团队内部使用（推荐）

```json
"relay": {
  "url": "wss://ai.chedian.cc/bridgeflow/ws/",
  "room_key": "team-zgc-2026-abc"
}
```

用公共中继，但换一个私有 room_key，确保只有你们团队收发消息。

### 场景 2：多台 PC 执行机

每台 PC 各自 `bridgeflow init`，生成独立的 `device_id` 和 `machine_code`，共用同一 `relay` 和 `room_key`。手机端可以看到所有 PC，分别绑定。

### 场景 3：自部署中继

```json
"relay": {
  "url": "wss://your-own-server.com/bridgeflow/ws/",
  "room_key": "internal-room"
}
```

同时修改 `web/pwa/config.js` 中的 `relayUrl` 保持一致。
