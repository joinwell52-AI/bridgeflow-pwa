## What's in this release

This is the first **end-to-end runnable** CodeFlow v2 cut. It bundles:

- `@codeflow/protocol` — 5 schemas (agent / task / review / session / skill) + `validate()`
- `@codeflow/runtime` — 8 subsystems / 14 modules:
  1. `RuntimeBootstrap` (sprint S2)
  2. `AgentRegistry` + `PersistentStore` (S2)
  3. `SessionManager` + `SessionStore` + `TranscriptWriter` (S3 phase B)
  4. `InboxWatcher` + `TaskParser` + `TaskDispatcher` + `StateHistoryWriter` (S3 phase C)
  5. `ReviewEngine` + `ReviewWriter` + `NeedsHumanGate` + `AgentStatusReconciler` (S4 phase D)
  6. `SkillRegistry` + `KernelDependencyValidator` + `MCPInjector` (S5 phase E)
- `codeflow-shell` — Layer-1 minimal entry (S6, this release):
  - `src/main.ts` (~120 lines)
  - `src/bootstrap.ts` (skill / agent fixture planters)
  - `src/sdk-factory.ts` (real-vs-fake SDK adapter chain)
  - `examples/hello-world/` (smoke-test demo)
  - `sea-config.json` + `pack.cmd` (Windows EXE)

### Test coverage

- 94 / 94 unit & integration tests pass (in `packages/codeflow-runtime`)
- 30 consecutive runs with **0 flakes** (post-S5 stabilisation)
- E2E demo (`examples/hello-world.ts` in runtime) verified
- Shell-level E2E (this release) — single drop-and-watch verified

### What you can do with this release

1. Double-click `CodeFlow-v0.1.0-rc.1.exe` (or `npm start` fallback) on Windows.
2. Drop a `TASK-*-XX-to-DEV.md` file into `~/.codeflow/v2/inbox/`.
3. Watch the shell stream the full governance loop in stdout:
   - dispatch → session start → session end → review → REVIEW-*.md persist → state_history append.
4. `Ctrl+C` to stop cleanly.

That's the whole v0.1 surface. No mobile pairing, no tray, no web panel, no real Cursor SDK calls. Those are v0.2.

## Known gaps & caveats

### Out-of-scope for v0.1 (per TASK-028 §二)

- ❌ Tray / web panel / relay bridge
- ❌ macOS / Linux packaging
- ❌ Self-startup registration / single-instance mutex
- ❌ Real `@cursor/sdk` integration (uses `InMemorySdkAdapter`)
- ❌ npm / PyPI / GitHub Release publish
- ❌ Modifications to any `@codeflow/protocol` schema (deferred to v0.2 sprint 0)

### Caveats

- **Node SEA is still maturing** (design doc §11.8 risk row 1). If `pack.cmd` fails, the official fallback is `npm start`. Node 22+ is required; Node 24+ recommended.
- **`Review.decision = "needs_human"`** + `human_approval` sub-structure are v0.1 concepts that **will be deprecated in v0.2** in favour of FCoP v1.0's Boundary capability ([FCoP issue #2](https://github.com/joinwell52-AI/FCoP/issues/2#issuecomment-4412811192)).
- **`InMemorySdkAdapter` synthetic settle**: the demo's `decision="needs_human"` is *expected* — it proves the verdict-parser fallback (TS-6.9) works, not a bug.

## v0.2 sprint 0 alignment plan

CodeFlow v0.2 sprint 0 will:

1. Replace `InMemorySdkAdapter` with the real `CursorSdkAdapter` once `@cursor/sdk` ships the doorbell primitive.
2. Drop `Review.decision="needs_human"` (and `human_approval`) — replaced by FCoP v1.0 Boundary capability.
3. Adopt FCoP v1.0's 7-abstraction model (currently 5 schemas in CodeFlow v0.1).
4. Add tray + web panel + relay bridge (deferred from S6).
5. Add macOS / Linux pack scripts.
6. First public npm release (`@codeflow/runtime@1.0.0` aligned to `fcop@>=1.0,<2.0`).

Tracking: PM is drafting `docs/design/v0.2-sprint0-roadmap.md`.
