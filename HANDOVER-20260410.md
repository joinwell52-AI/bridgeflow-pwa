**版本**：Desktop v2.9.12 / PWA v2.0.3

## 一、三条工作线

```
本地开发  →  改代码
PWA 发布  →  py -3 _deploy_pwa.py
GIT 备份  →  git push backup main
```

### 仓库说明

| 仓库 | 地址 | 用途 |
|------|------|------|
| `codeflow-pwa`（公开） | github.com/joinwell52-AI/codeflow-pwa | PWA 静态文件，GitHub Pages 发布手机端 |
| `codehouse`（私有） | github.com/joinwell52-AI/codehouse | 完整代码备份（`git push backup main`） |

> `codeflow-pwa` 原名 `bridgeflow-pwa`，已改名。  
> `origin` = codeflow-pwa，`backup` = codehouse。

---

## 二、PWA 发布流程

```
1. 改 web/pwa/ 里的文件
2. 升版本号：
   - web/pwa/config.js  → appVersion: "x.x.x"
   - web/pwa/sw.js      → // build: YYYYMMDD-x.x.x
3. 运行：py -3 _deploy_pwa.py
```

- Token 存放在 `.github_token`（已列入 `.gitignore`，不进 git）
- `codeflow-pwa` 仓库只存 5 个 PWA 文件，不含主仓库代码
- 手机刷新 PWA 后在"我的"看版本号确认更新

---

## 三、Desktop 打包流程

```
1. 改代码
2. 升版本号：
   - codeflow-desktop/main.py      → VERSION = "x.x.x"
   - codeflow-desktop/web_panel.py → _VERSION = "x.x.x"
3. 打包：
   cd codeflow-desktop
   pack.cmd
4. 产物：codeflow-desktop/dist/CodeFlow-Desktop.exe
```

## 四、当前版本状态

### Desktop v2.9.12
- 嵌入 Cursor：`Ctrl+Shift+B` 打开 Browser Tab → 粘贴 URL（不再走命令面板）
- Cursor 已开着再启动 CodeFlow：等焦点稳定后嵌入
- 巡检重试：`agent_busy` 不消耗重试次数
- 单实例：固定端口 `18765`，固定 mutex `CodeFlowDesktop_18765`

### PWA v2.0.3
- 巡检轨迹：通过 WebSocket 事件 `patrol_trace`/`patrol_state` 接收，不用 HTTP 直连
- PC 在线后自动更新：任务列表、今日统计、巡检状态、巡检轨迹
- 点"刷新"按钮触发 PC 推送最新数据

## 七、注意事项

- 修改含中文的文件（`.html` / `.md` / `.py`）必须用 Python 读写，禁止 PowerShell `-replace`
- 不要把真实 `room_key` 写到公开示例配置
- 中继限制单条消息 8KB
- `docs/agents/tasks/` 下已有任务文件只追加，不修改
