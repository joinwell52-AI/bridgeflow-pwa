## §二 迁移范围（精确 — PM 已做 path 三件套核对）

### §2.1 物理文件 git mv

```powershell
git mv docs/agents fcop
# 如果 docs/internal/ 存在 → 也 git mv 到 fcop/internal/ （PM 见过 emergence-log.md 在 docs/internal/）
# 注：docs/design/ 是设计文档，**不动**（不属于 fcop workspace 范畴）
```

**预期变更**：172 文件 rename（git rename detection 保 file history）。

**OPS 自查**：
```powershell

```

### §2.2 代码内硬编码 path 替换（PM 已 ripgrep 标记 — OPS 自查实际数量）

PM ripgrep 预期命中点（OPS 必须以实际 ripgrep 结果为准）：

| 文件 | 当前 | 改后 |
|---|---|---|
| `packages/codeflow-runtime/src/Runtime.ts` | `workspaceDir: "docs/agents"` 之类 | **删 workspaceDir 参数**（让 fcop auto-detect 命中 `<root>/fcop/`）|
| `codeflow-shell/src/main.ts` | banner 显示 `docs/agents/...` 字样 | 改 `fcop/...` |
| `codeflow-shell/.env.example` | `CODEFLOW_INBOX_DIR=docs/agents/tasks` 之类 | 改 `fcop/tasks` 或加注释「fcop auto-detect」|
| `packages/codeflow-runtime/src/scheduler/InboxWatcher.ts` | 默认 watch dir | 改 `fcop/tasks` |
| `packages/codeflow-runtime/src/_internal/atomic-write.ts`（如有）| paths 默认值 | 改 |
| `packages/codeflow-runtime/src/Runtime.ts`（fcopClient 创建处）| `FcopProjectClient.create({ workspaceDir: "docs/agents" })` | **删 workspaceDir** |

**OPS 自决**：用 ripgrep 找 `docs/agents` + `docs\\agents` 在 `.ts/.js/.json/.env*` 中的全部命中，逐一审查 + 替换或删除。

### §2.3 文档 / 规则文件 path 引用更新

| 文件类别 | 处理 |
|---|---|
| `.cursor/rules/*.mdc`（数十处 `docs/agents/tasks/...` 字样）| 改 `fcop/tasks/...` |
| `docs/design/codeflow-v2-on-fcop-sdk.md`（如引用 path）| 改 |
| `README.md` + `docs/codeflow-overview.md` 等公开文档（如引用 path）| 改 |
| `private/` 下的 HANDOVER 文档 | OPS 自决：可改也可不改（private 不在 release 范围）|

### §2.4 历史 task/report body 内引用 — **保持不动**

| 来源 | 处理 |
|---|---|
| 历史 task body 里 `docs/agents/tasks/TASK-...` 字符串引用 | **保持不动** |
| 历史 review/report body 内同上 | **保持不动** |

**理由**：`codeflow-project.mdc` 严禁事项「不要修改 docs/agents/tasks/ 和 reports/ 下已有的任务文件（只追加）」**严守**。git mv 让物理路径变化，但**文件内引用字符串保持历史原貌** — 这是不可变历史的语义。

### §2.5 fcop 元数据文件

| 文件 | 处理 |
|---|---|
| `docs/agents/state.json`（fcop sequence-generator state — 如存在）| `git mv` 到 `fcop/state.json` |
| `docs/agents/.fcop-config`（如存在）| 同上 |

OPS 自查：`ls docs/agents/ -Force` 找隐藏 metadata 文件。
