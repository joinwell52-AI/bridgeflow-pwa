## 范围（in-scope）

### 必交付 1：`PersistentStore` 实现（`packages/codeflow-runtime/src/registry/PersistentStore.ts`）

按 `crash-recovery.md` **决策 1**（atomic-write + fsync）实现：

- `loadAll(): Promise<AgentRecord[]>` — 读 `agents.json`；文件不存在 = 返回 `[]`；解析失败 = HARD FAIL（throw + 不污染状态）
- `saveAll(records: AgentRecord[]): Promise<void>` — 写流程必须**严格按以下顺序**：
  1. `fs.writeFile(`${path}.tmp`, JSON.stringify(records, null, 2))` （写临时文件）
  2. `fs.rename(`${path}.tmp`, path)` （atomic rename，POSIX/NTFS 都保证原子性）
  3. `fs.fsync(parent_dir_fd)` （父目录元数据落盘——这一步在 Linux 上必要；Windows 上无害且兼容）
- 单 record CRUD（`upsert(record): Promise<void>` / `removeById(agentId): Promise<void>`）= 内部读全表 + 修改 + saveAll
- 错误处理：写失败 = throw + 不污染原文件（因为是 write-temp + rename 模式，原文件直到 rename 那一刻才被替换）
- **不在本必交付里**：watching、并发锁、批量 batching（决策 1 已明确 batch 模式被否决）

### 必交付 2：`AgentRegistry` 6 方法的 method body 实现（`src/registry/AgentRegistry.ts`）

去掉 `throw new Error("[S2 skeleton] ...")`，按 [TASK-002 必交付 2](./TASK-20260509-002-PM-to-DEV.md) 列出的 6 方法签名 + JSDoc 不变量实现：

| 方法 | 实现要点 |
|---|---|
| `register(agentSpec)` | (a) 入参 `validate(agentSchema, agentSpec)`，失败 = throw `ValidationError`；(b) `agentSpec.layer === "admin"` = throw `LayerViolationError`（兑现 §3.2 layer 约束）；(c) 调 SDK `Agent.create(...)` 拿 `sdk_agent_id`；(d) 构造 `AgentRecord`（status: "idle"）；(e) `_store.upsert(record)`；(f) 返回 record |
| `resume(agentId)` | (a) `_store.loadAll()` 找 record，没找到 = throw `AgentNotFoundError`；(b) 调 SDK `Agent.resume(record.sdk_agent_id)` —— resume 失败的 error message 必须可读（包含 sdk_agent_id 和原始错误）+ 不修改 record；(c) 更新 `record.last_active_at` / `runtime_last_reconciled_at` 字段（runtime 私有字段，按 REPORT-002 §四已定义）；(d) `_store.upsert`；(e) 返回 record |
| `list(filter?)` | 直接 `_store.loadAll()` 后用 filter 过滤；filter 字段全部可选；filter 默认空 = 全部 |
| `get(agentId)` | `_store.loadAll()` 后 find；不存在 = `null`，**不抛异常** |
| `updateRuntimeBinding(agentId, runtime)` | (a) get record；(b) 如 record.runtime === runtime 则 no-op；(c) 否则更新字段 + `_store.upsert` —— **不在本任务自动触发 resume**（避免副作用串联，留给上层显式 call resume） |
| `markFailed(agentId, error)` | (a) get record；(b) 写 `record.status = "error"` + `record.runtime_failure = { failed_at, reason }`（runtime 私有结构，按 REPORT-002 §四已定义）；(c) `_store.upsert` |

**关键不变量必须在实现里 enforce**：
- 所有 *写* 操作前必须 ajv-validate（用 `@codeflow/protocol` 的 validator）
- 写失败 = 不污染 `agents.json`（PersistentStore 的 atomic-rename 模式自动保证）
- `layer === "admin"` 的 register 必须在调 SDK 之前 reject（不能等 SDK 那边 fail）
- 所有错误用 named class（`ValidationError` / `LayerViolationError` / `AgentNotFoundError` / `RegistryWriteError` 等）

### 必交付 3：`RuntimeBootstrap` 类（`src/registry/RuntimeBootstrap.ts`，新文件）

按 `crash-recovery.md` **决策 2** + **决策 3** 实现启动 reconciliation 流程：

```ts
class RuntimeBootstrap {
  constructor(opts: { store: PersistentStore; sdk: AgentSdkAdapter });
  async run(): Promise<ReconciliationReport>;
}
```

启动顺序：

```
1. _store.loadAll() → records[]
   ├─ 文件不存在        → records = []，跳到 step 3
   ├─ 文件解析失败      → throw RuntimeBootstrapError("agents.json corrupted: ..."); process.exit(1) 由调用方决定
   └─ 解析成功           → 进 step 2

2. SDK.list() → sdkAgentIds (Set<string>)
   for each record in records:
     ├─ record.sdk_agent_id ∈ sdkAgentIds → 调 SDK.resume(sdk_agent_id)
     │     ├─ resume 成功 → record.runtime_last_reconciled_at = now ; report.success.push(...)
     │     └─ resume 失败 → markFailed(record.agent_id, "resume after crash failed: <原始 error>") ;
     │                     report.failed.push(...)
     └─ record.sdk_agent_id ∉ sdkAgentIds → 决策 3 场景 X
                              → markFailed(record.agent_id, "orphaned: SDK no longer recognizes sdk_agent_id") ;
                              → report.orphaned.push(...)

   for each foreign sdkAgentId in sdkAgentIds where ∄ record:
     → 决策 3 场景 Y
     → report.foreign.push(sdkAgentId)（不接管，仅记录）

3. 输出 reconciliation summary 到 stdout：
   ✅ 4 success / ⚠️ 1 failed / 🪦 1 orphaned / 👻 1 foreign

4. 返回 ReconciliationReport
```

**关键不变量**：
- `RuntimeBootstrap.run()` 期间**禁止**调用方触发 `register()`（race 防御）—— 提供一个内部 `_isBootstrapping: boolean` 标志，AgentRegistry 在 register 入口处检查；如 bootstrapping = throw `RuntimeNotReadyError`
- 单个 record 的 reconcile 失败**不阻断整个 run**——批量收集后返回 report
- `agents.json` 解析失败 = HARD FAIL（决策 2 末尾已明确"不允许半启动状态"）
