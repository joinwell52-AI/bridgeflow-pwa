# TASK-20260511-002：repo 根 `.gitignore` 加 `.smoke-*` + 清理 7 个 stale scratch dirs

## 一、TL;DR

QA-014 在 repo 根目录留下 7 个 `.smoke-qa014-*` scratch 目录（其中 4 个**带尾随空格**的特殊命名）。`codeflow-shell/.gitignore` 已有 `.smoke-*` 规则，但 **repo 根 `.gitignore` 没有** — 这些目录污染 `git status` 显示且有被误 `git add .` 提交的风险。

本任务两步：
1. repo 根 `.gitignore` 新增一行 `.smoke-*/`（与 codeflow-shell 子树同款规则）
2. 清理工作树残留的 7 个 scratch 目录（注意带空格命名需 PowerShell 显式 quote）

## 二、当前 `git status --short` 与本任务相关项

```text
?? .smoke-qa014-20260511-080830/
?? ".smoke-qa014-claude /"          ← 注意尾随空格
?? .smoke-qa014-claude/
?? ".smoke-qa014-cs45 /"            ← 注意尾随空格
?? .smoke-qa014-cs45/
?? ".smoke-qa014-default /"         ← 注意尾随空格
?? .smoke-qa014-default/
```

共 7 个目录。带尾随空格的 4 个是 QA-014 用 PowerShell 调用时未正确 trim 引号导致，是 Windows 文件系统 bug surface（NTFS 允许尾随空格，但大多数工具不显示）。

`codeflow-shell/.gitignore` line 25 已有 `.smoke-*` — 仅对 `codeflow-shell/` 子树生效，不覆盖 repo 根。

`.gitignore` repo 根 line 11-12 有：
```text
*.log
*.tmp
```

但缺 `.smoke-*` 规则。

## 三、必交付清单

### 3.1 改 repo 根 `.gitignore`

在合适位置（建议第 "运行时" 段后，line 62 附近）新增：

```text
# ── Smoke / dev-test scratch dirs at repo root (PM TASK-20260511-002) ──
# DEV / QA self-tests sometimes drop `.smoke-*` scratch directories at
# the repo root (e.g. `.smoke-qa014-20260511-080830/`). codeflow-shell
# subtree already excludes them; this line covers the repo root.
.smoke-*/
```

注意：用 `.smoke-*/`（带 `/`）而不是 `.smoke-*` — 仅排除目录，不误伤未来可能出现的 `.smoke-config.json` 之类文件。

### 3.2 清理 7 个 stale scratch dirs

PowerShell 命令（**OPS 必须在 stage 前完成**，避免 `git add .` 误捕）：

```powershell
# 不带空格的 3 个
Remove-Item -Recurse -Force ".smoke-qa014-20260511-080830"
Remove-Item -Recurse -Force ".smoke-qa014-claude"
Remove-Item -Recurse -Force ".smoke-qa014-cs45"
Remove-Item -Recurse -Force ".smoke-qa014-default"

# 带尾随空格的 3 个（注意引号必须紧贴空格）
Remove-Item -Recurse -Force ".smoke-qa014-claude "
Remove-Item -Recurse -Force ".smoke-qa014-cs45 "
Remove-Item -Recurse -Force ".smoke-qa014-default "
```

**若某目录有进程占用**（如 OPS-014 §四 §10 提到的 `shell.log` 被进程持有 EPERM 错误）：
- 第一次 EPERM → 等 30 秒重试
- 第二次 EPERM → 写 REPORT 列出占用目录，**不强杀进程**，PM 决定后续

### 3.3 Safety HARD GATE（OPS 必须做）

- 删除前：`git ls-files --error-unmatch .smoke-qa014-20260511-080830` 等命令应**全部 fatal**（证明这些目录未 tracked，安全删除）
- 删除前/后：`git diff` 在被改的 `.gitignore` 上仅显示新增的 5-7 行
- stage `git add .gitignore` 后：`git diff --cached` 仅显示 `.gitignore` 一个文件
- **不**在本次 commit 中携带：DEV-15 的代码改动 / DEV-014 的任何 untracked task/report 文档

### 3.4 commit + 推送

仅一个 commit（**不**双 commit）：

```text

```

完整 commit body 模板（OPS 自由调整措辞）：

```text

Refs: TASK-20260511-002, REPORT-20260510-014-OPS-to-PM (§四 第 10 项),
```
