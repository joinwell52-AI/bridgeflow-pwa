### 3.2 主交付 2：5 个核心 fcop 调用 demo（必做）

在 `hello-fcop.ts` 同目录写 `demo-fcop-api.ts`，**端到端**演示：

```typescript
import { python } from 'pythonia';

const fcop = await python('fcop');

// (1) Project 初始化
const project = await fcop.Project('D:/temp/codeflow-spike-project', { strict$: false });

// (2) 写一张任务（fcop 自动生成 frontmatter）
const taskFilename = await project.write_task(
  'PM',           // sender
  'DEV',          // recipient
  'spike test',   // subject
  'just a test',  // body
  { priority$: 'P0', risk_level$: 'low' }  // v1.1 新参数
);

// (3) 列任务
const tasks = await project.list_tasks({ status$: 'open' });
console.log('open tasks:', tasks.length);

// (4) 写一张 review（v1.1 新工具）
const reviewFilename = await project.write_review(
  'QA',                  // reviewer_role
  'task',                // subject_type
  taskFilename,          // subject_ref
  'needs_human',         // decision (v1.1 ADR-0025 第 5 值)
  'just spiking',        // rationale
  null                   // required_changes
);

// (5) 标记 human approved（v1.1 新工具，ADMIN-only ACL）
await project.mark_human_approved(
  reviewFilename.replace('.md', ''),   // review_id
  'ADMIN',                              // approver
  'approve',                            // decision
  'cli',                                // channel
  'spike approval'                      // comment
);

console.log('all 5 calls passed');
await python.exit();
```

期望输出：
```
open tasks: 1
all 5 calls passed
```

**重点观察**：
- 5 调用是否都通过（任何 fail 就是 surprise）
- 每个调用延迟（同进程应 < 50ms / 调用）
- 文件实际生成的位置 + frontmatter 是否符合 fcop v1.1 schema

# P4 Schema 映射：CodeFlow v0.1 → fcop@1.1.0

## 一对一映射
| CodeFlow v0.1 | fcop@1.1.0 | 差异 |
|---|---|---|
| agent.schema (TS interface) | agent.schema.json | ... |

## fcop 独有 (CodeFlow 之前没有)
| fcop schema | 用途 | CodeFlow 是否需要 |
|---|---|---|
| boundary.schema.json | 边界 / 隔离 | ... |
| encoding.schema.json | 编码协商 | ... |
| ipc-envelope.schema.json | IPC 信封 | ... |
| event.schema.json | 事件 | ... |
| failure.schema.json | 失败语义 | ... |

## CodeFlow 独有需要迁移到运行时
| CodeFlow 字段 | 处置 |
|---|---|
| Agent.layer (worker/governance/admin) | fcop v1.1 ADR-0023 已对齐，直接用 fcop |
| Task.priority (P0-P3) | fcop v1.1 已有 priority + 新 risk_level；并存 |
| Review.decision = needs_human | fcop v1.1 ADR-0025 第 5 值，直接用 fcop |
