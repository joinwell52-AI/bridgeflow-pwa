## v2.12.0 主要变更

### Desktop

- `_build_dashboard`：修复 `status` → `progress` 字段兼容（报告文件 `status: completed` 不再显示"等待中"）
- `_build_dashboard`：`sender` 字段自动回退到 `reporter` / `author`，修复发起人显示问题
- `_build_dashboard`：`replied_threads` 只统计 `recipient` 含 ADMIN 的 reports
- `_read_team_info`：新增 `leader` 字段，按模板默认（mvp-team→MARKETER 等）
- `cursor_cdp.py`：新增 `elToMd()` 将 Cursor DOM 转为 Markdown，通过 `msg.md` 字段传输（最多 3000 字节）

### PWA

- 工作实况气泡：Markdown 渲染（加粗/列表/代码块），用户/Agent 气泡左右对齐区分
- 工作实况：优先使用完整 `msg.md` 渲染，不再截断
- 团队卡片：主控角色（队长）显示金色 ★ 星标
- PC 重连自动刷新：PC 上线后 2s/5s/12s 递增重试拉取 dashboard
- 顶部状态栏：PC 在线时显示小 ↻ 刷新图标，可手动触发
- 已完成任务列表：只显示 `recipient` 含 ADMIN 的 reports（不含内部报告）
- 任务状态：从 YAML `status` 字段读取，报告文件正确显示"已完成"

## 关于推广页版本号

**不会自动更新**，是硬编码在 HTML 里的。每次发版后需手动改：

- `promotion/index.html` 第 142-143 行（shields.io badge）
- `docs/promotion/index.html` 同上
- `README.md` 第 11-12 行（shields.io badge）

**建议改进**：未来可在推广页 JS 里通过 GitHub API 动态读取最新 Release 版本号，实现自动更新。
